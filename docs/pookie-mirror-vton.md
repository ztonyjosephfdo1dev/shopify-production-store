## 1. Executive Summary

**Pookie Mirror** is a customer-facing virtual try-on feature embedded in the Shopify storefront. Customers upload a selfie, pick garments from their cart, and see themselves wearing those garments in 6 professional editorial poses — all generated by Gemini AI in a single API call.

**Key stats:**
- **Cost:** $0.00 per try-on (Gemini free tier)
- **Latency:** 15-40 seconds for 6 editorial panels
- **Max garments:** 3 per try-on (any category combo)
- **Session limit:** 5 try-ons per session
- **Hosting:** GCP Cloud Function (₹0/month — free tier)

## 3. Frontend — Pookie Mirror Drawer

### Component Files
- `theme/snippets/pookie-vton-room.liquid` — ~980 lines (HTML + JS)
- `theme/assets/pookie-vton-room.css` — ~750 lines
- `theme/assets/pookie-vton-music.mp3` — background audio for result screen
- Included in `theme/layout/theme.liquid` site-wide

### State Machine

```
builder  →  loading  →  result
   ↑                       │
   └───── try again ───────┘
```

### FAB Button
- Floating action button (✨) visible on every page
- Shows cart badge count
- Opens the full-screen drawer (slides up on mobile, sidebar on desktop)
- Triggered by: FAB tap, or `pookie-mirror:open` custom event from product page "Try This On" button

### Builder Screen

#### Photo Gallery (max 3 photos)
- Upload via file input → resize to max 1536px → compress (WebP with JPEG fallback, target ≤500KB)
- Stored in `localStorage` — persists across page navigations
- Select/remove photos, last-selected persisted
- Photos converted to JPEG before API call (Gemini compatibility)

#### Outfit Builder (max 3 garments)
- Populates from Shopify cart (`/cart.js`)
- For each cart item, fetches `/products/<handle>.json` to find:
  - **Garment reference image**: image with alt text `"garment photo"` or filename containing `"garment-real"`
  - Stored as `product_image_3d` URL
- Category read from cart line item property: `item.properties._garment_category` (fallback: `full_body`)
- No category filtering — all cart items appear in a flat horizontal tray
- Category displayed as badge: 👕 Top / 👖 Bottom / 👗 Full Outfit / 👔 Coord Set

#### Smart Hints
| Outfit State | Hint |
|---|---|
| 1 coord set | "✨ Coord set — complete matching outfit ready!" |
| 1 bottom only | "💡 Tip: Add a matching top for a complete look" |
| 1 top only | "💡 Matching bottom & accessories will be auto-styled" |
| 3 garments (max) | "✨ Full outfit ready — let's see your look!" |

### Cart Tray
- Horizontal-scrollable strip of all cart items
- Tap to add garment to outfit (gray out when added)
- Intercepts `fetch()` and `XMLHttpRequest` for `/cart/add|change|update` to detect cart changes in real-time

### Generation Flow
1. Validates: photo selected + ≥1 garment + tryons remaining
2. Converts WebP photos to JPEG for Gemini compatibility
3. Re-fetches `product_image_3d` for any garment still missing it
4. Sends single `POST ?action=tryon_grid` with:
   ```json
   {
     "person_image": "<base64 JPEG>",
     "garments": [
       {
         "image_url": "https://cdn.shopify.com/.../hero.jpg",
         "garment_brief": "Red cotton kurti, half sleeves",
         "garment_category": "upper_body",
         "garment_design_details": "...",
         "product_image_3d": "https://cdn.shopify.com/.../garment-real.jpg"
       }
     ],
     "tryon_mode": "customer_editorial",
     "quality": "medium"
   }
   ```

### Loading Screen
- Rotating fashion quotes (4-second interval)
- Progress animation
- "This usually takes 15-30 seconds" message

### Result Screen
- Full-screen animated slideshow of 6 panels
- 8 Ken Burns / slide transition effects: zoomIn, zoomOut, panLeft, panRight, panUp, slideLeft, slideRight, fadeZoom
- Auto-plays background audio (`pookie-vton-music.mp3`) with toggle
- Dot navigation + counter (1/6, 2/6, etc.)
- Download button (saves current panel as image)
- "Try Again" button → returns to builder screen

### Session Limit
- Max 5 try-ons per session via `sessionStorage`
- Counter displayed in drawer

## 4. Backend — API Endpoints

Base URL: `https://asia-south1-pookie-style-automation.cloudfunctions.net/create-product`

### `?action=tryon_grid` (Primary — Live)

| Field | Details |
|---|---|
| **Purpose** | Generate 6-panel customer-editorial try-on grid |
| **Input** | `person_image` (base64 JPEG), `garments[]` (up to 3), `quality`, `tryon_mode` |
| **Process** | Parse request → Gemini generates 2×3 grid → math_crop → compress 6 panels |
| **Output** | `{ "panels": ["<b64>", ...], "phase": "grid" }` |
| **Models** | gemini-3.1-flash-image-preview → gemini-2.5-flash-image |
| **Cost** | $0.00 |

### `?action=tryon_fast` (Available — Not Used by Current Frontend)

| Field | Details |
|---|---|
| **Purpose** | Single inpainting edit — swap clothing, preserve face/body exactly |
| **Input** | Same as tryon_grid |
| **Process** | Parse request → Gemini inpainting → single image |
| **Output** | `{ "panels": ["<b64>"], "phase": "fast" }` |
| **Models** | gemini-3.1-flash-image-preview → gemini-2.5-flash-image |
| **Cost** | $0.00 |

### `?action=tryon` (Legacy Alias → Routes to tryon_fast)

### Garment Object Schema
```json
{
  "image_url": "https://cdn.shopify.com/.../hero.jpg",
  "image_base64": "...",
  "garment_brief": "Red cotton kurti, half sleeves",
  "garment_category": "upper_body | lower_body | full_body | coord_set",
  "garment_design_details": "V-neckline, floral embroidery, short sleeves...",
  "product_image_3d": "https://cdn.shopify.com/.../garment-real.jpg"
}
```

### Request Parsing (`_parse_tryon_request()`)
1. Decodes `person_image` from base64
2. For each garment:
   - Fetches image from `image_url` (or decodes `image_base64`)
   - Fetches `product_image_3d` URL (clean garment reference)
3. Returns: `person_bytes, garment_images, garment_briefs, garment_categories, garment_design_details_list, garment_3d_images, quality, tryon_mode`

## 5. Backend — Generation Logic

### 5.1 Phase 2: `generate_tryon_grid()` (Current Live VTON)

**Purpose:** Generate a 2×3 customer-editorial grid — the customer wearing the garment(s) in 6 editorial poses.

**Key principles:**
- All 6 panels feature the uploaded **customer** (no 3D/mannequin panels in VTON mode)
- Customer's face, hair, body, skin tone, proportions preserved **exactly** across all panels
- Garment details (color, pattern, print, text, logo) reproduced pixel-perfectly

**6 Panel Descriptions (Customer-Editorial Mode):**

| Panel | Description |
|---|---|
| 1 | Hero portrait — front-facing confidence shot, waist-up, premium beauty lighting |
| 2 | Soft 3/4 portrait angle, natural hand pose, face + upper body clearly visible |
| 3 | Lifestyle standing shot, full body, warm candid mood, premium campaign feel |
| 4 | Detail close-up with face visible, highlight garment texture/neckline/sleeve |
| 5 | Movement shot, elegant walking pose, balanced framing, flattering silhouette |
| 6 | Signature editorial frame, confident final pose, high-fashion campaign quality |

**Face Swap Prevention:**
- When `product_image_3d` (clean garment photo) is available: uses **ONLY** that as garment reference — never sends the hero product image (which contains an AI-generated model whose face could contaminate the output)
- When no clean ref is available: crops top 25% of product photo to remove AI model face
- Explicit prompt instructions: "The person in the garment photo is NOT the customer. IGNORE that person entirely."
- Customer photo explicitly labeled: "the ONLY person in this generation"

**Grid Validation:**
- Checks aspect ratio after generation — detects 2×2 grid (ratio < 1.25) vs expected 2×3
- Retries once with emphatic instructions if 2×2 detected
- Each model gets up to 2 attempts

**Model Cascade:** gemini-3.1-flash-image-preview → gemini-2.5-flash-image

### 5.2 Phase 1: `generate_tryon_single()` (Available, Not Currently Called)

**Purpose:** Single-image inpainting — change ONLY clothing on customer's real photo while preserving face, hair, body, and pose exactly.

**Key features:**
- Preserves face, hair, body shape, pose, skin tone — **absolute #1 priority**
- Changes only clothing to the garment(s) from product photos
- Upgrades background to editorial quality (studio, café, boutique, urban street)
- Auto-generates complementary accessories based on garment category
- Returns 1 high-quality image

## 6. Multi-Garment & Layering Support

### Slot Mapping

Each garment category gets a slot description in the prompt:

| Category | Slot Description |
|---|---|
| `upper_body` | "on upper body" |
| `lower_body` | "on lower body" |
| `full_body` | "as complete outfit" |
| `coord_set` | "as coordinated set (matching top + bottom)" |

### Layering Logic

| Scenario | Behavior |
|---|---|
| Multiple `upper_body` garments | First = base layer, subsequent = outer layer |
| `full_body` + `upper_body` | Upper piece layered OVER the full outfit |
| `coord_set` + `upper_body` | Upper piece layered OVER the coord set |
| Multiple `lower_body` garments | Each listed separately (no layering) |

### Accessory Auto-Styling

The backend automatically generates complementary accessory instructions based on the garment category combination:

| Combination | Auto-Styled Accessories |
|---|---|
| `coord_set` alone | Matching footwear, minimal accessories — set is the star |
| `full_body` alone | Matching footwear, small bag/clutch, dainty earrings |
| `upper_body` + `lower_body` | Matching footwear, minimal accessories |
| `upper_body` alone | Complementary bottom wear, footwear, dainty jewelry |
| `lower_body` alone | Complementary top, footwear, minimal accessories |
| `full_body` + `upper_body` | Layered look — top over dress, matching footwear |
| Multiple `upper_body` garments | Layered visible — inner + outer, footwear, minimal |

## 8. Image Processing

### Grid Crop (`math_crop_grid()`)
- Pure mathematical split: width ÷ 2 columns, height ÷ 3 rows
- Trims 2px from each panel edge to remove border artifacts
- Used for VTON grids (no divider lines expected)

### Compression (`compress_for_shopify()`)
- Each panel compressed to ≤400KB JPEG
- Resizes to max 2048px on any side
- Progressive quality reduction: 92 → 55, stepping by 3

## 9. Data Flow — From Cart to Try-On

### How Garment Data Flows to VTON

1. **Product creation** (admin): AI detects garment type → maps to category → stored as `pookie.garment_category` metafield. Garment brief, design details, real garment URL, 3D image URL also stored as metafields.

2. **Add to cart** (customer): Shopify line item properties include `_garment_category`, `_garment_brief`, `_garment_design_details` (set via product page JS or theme customization).

3. **Pookie Mirror** (customer): Reads cart → for each item:
   - `garment_category` from `item.properties._garment_category` (fallback: `full_body`)
   - `garment_brief` from `item.properties._garment_brief` (fallback: product title)
   - Fetches `/products/<handle>.json` to find `product_image_3d` (clean garment photo by alt text)

4. **API call**: All garment data sent in `garments[]` array → backend uses for slot mapping, accessories, and prompt construction.

## 10. Prompt Architecture

### Identity Preservation (Priority #1)
The prompt enforces complete identity preservation:
- Exact face features (eyes, nose, mouth, jawline, expression)
- Exact hair (color, style, length, texture, parting)
- Exact body (shape, proportions, skin tone)
- Same apparent age and ethnicity
- "Think of this as: this EXACT person walked into your studio and you photographed her"

### Garment Fidelity (Priority #2)
- Pixel-perfect reproduction of garment from reference photos
- Text reproduction rules (exact spelling, font, position, color)
- Logo/graphic reproduction rules (exact shape, size, color, position)
- Pattern/stripe rules (count, width, spacing, colors, direction)
- Garment structure rules (collar, neckline, sleeve, fit, hem, color)

### Design Details Checklist
When `garment_design_details` is available (from product creation AI analysis), a detailed reproduction checklist is injected into the prompt with per-element verification instructions.

## 11. Error Handling & Fallbacks

| Failure | Handling |
|---|---|
| Gemini model 1 fails | Falls back to model 2 (gemini-2.5-flash-image) |
| Both Gemini models fail | Returns error to frontend — "Try again" button |
| Rate limit (429) | Waits 15 seconds, then retries |
| 2×2 grid instead of 2×3 | Detects via aspect ratio, retries once with emphatic instructions |
| No image in response | Tries next model in cascade |
| Frontend: no photo selected | Validation prevents generate button |
| Frontend: no garments selected | Validation prevents generate button |
| Session limit reached | Shows "limit reached" message |
| Clean garment ref missing | Falls back to cropped product photo (top 25% removed) |
